***работа с локальным и удаленным репозиторием

--создаем папку и переходим в нее в командной строке
git init	-инициализируем локальный репозиторий
git remote -v	-просмотр подключенных удаленных репозиориев, убеждаемся что ничего не подключено

--на сайте GitHub создаем удаленный репозиторий и копируем ссылку на него
--подключаем локальный репозиторий к удаленному
git remote add project https://github.com/fedotoff144/NoteProject.git	-подключаем удаленный репозиторий к 
									локальному, то есть теперь они связаны и 
									локальные изменения будут пушиться туда

--делаем локальный коммит после инициализации
git status			-проверяем статус изменений в локальном репозитории
git add .			-добавляем все файлы и папки к отслеживанию
git commit -m 'init commit'	-делаем коммит и смотрим на какой ветке находимся, ее и будем пушить

git push -u project master	-пушим локальные изменения в удаленный репозиторий


***
git fetch			-стягиваем все удаленные изменния без слияния
git log project/main main	-смотрим лог и сравниваем коммиты (и состояние HEAD) удаленного и локального репозиториев
git diff main project/main	-смотрим разницу кода между репозиториями
git merge project/main		-делаем слияние удаленного репозитория с локальным
или просто не делаем мердж и на локальной ветке все остается без изменений


*git pull origin branchname --allow-unrelated-histories -если есть несвязная история то ее можно проигнорировать
